‚úÖ Day 3: Types of Machine Learning

  1. Supervised Learning

    • Labeled data (input ‚Üí output)
    • Examples: Linear Regression, Classification

  2. Unsupervised Learning

    • No labels (find structure)
    • Examples: Clustering, Dimensionality Reduction

  3. Reinforcement Learning

    • Agent learns by interacting with environment
    • Examples: AlphaGo, Self-driving cars

‚úÖ Day 4: Batch vs Online Learning

üì¶ Batch (Offline) Learning:

  • Trains on entire dataset
  • Slower updates, good for stable systems

üåê Online Learning:

  • Learns incrementally (one sample at a time)
  • Best for real-time data or streaming
Type Memory Speed Use Case
Batch High Slow Static systems
Online Low Fast Real-time systems

‚úÖ Day 5: Online Learning Continued

  • Concept Drift: When data distribution changes over time (e.g., spam filters)
  • Out-of-core learning: Learning from data that doesn‚Äôt fit in memory
  • Learning Rate: Controls step size in learning

‚ö†Ô∏è Risk: Online models can be unstable if learning rate is too high.

‚úÖ Day 6: Instance-Based vs Model-Based Learning

üìå Instance-Based:

  • Memorizes training examples (lazy learning)
  • Example: K-Nearest Neighbors

üìå Model-Based:

  • Learns a mathematical model (generalizes)
  • Example: Linear Regression, Decision Trees
Aspect Instance-Based Model-Based
Learning Time Low High
Prediction Time High Fast
Accuracy Local Global

‚úÖ Day 24: Feature Scaling ‚Äî Standardization

‚öñÔ∏è Why Needed:

  • ML algorithms like KNN, SVM, Logistic Regression, and Gradient Descent are sensitive to feature scale.

üìê Standardization:

Formula:
                 \( z = \frac{x - \mu}{\sigma} \)

  • Mean (\(\mu\)) = 0
  • Standard Deviation (\(\sigma\)) = 1

üì¶ Sklearn Code:

from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

üß™ When to Use:

  • Features are normally distributed
  • Algorithms using distance or gradient descent

‚úÖ Day 25: Feature Scaling ‚Äî Normalization

üìê Normalization:

Transforms features into a bounded range ‚Äî typically [0, 1]

üî¢ Min-Max Scaling:

Formula:
                 \( x_{\text{norm}} = \frac{x - x_{\min}}{x_{\max} - x_{\min}} \)

from sklearn.preprocessing import MinMaxScaler
scaler = MinMaxScaler()
X_scaled = scaler.fit_transform(X)

üî∏ Other Types:

Type When to Use
MaxAbsScaler Sparse data (keeps zero entries)
RobustScaler Data with outliers
Mean Normalization Not commonly used in sklearn
MinMaxScaler When bounded output is expected (e.g., pixel values)

üí° When to Use Normalization:

  • Algorithms like KNN, Neural Networks
  • Models where magnitude matters

‚úÖ Day 26: Encoding Categorical Variables ‚Äì Ordinal & Label Encoding

üéØ Why Encoding Is Needed:

ML models only work with numerical inputs, so we must convert categorical variables (like 'Male', 'Female') into numbers.

üî∑ 1. Ordinal Encoding

  • Assigns a rank or order to each category
  • Use when categories have a logical sequence

Example:

education = ['High School', 'Bachelor', 'Master', 'PhD']
Encoded as ‚Üí [1, 2, 3, 4]

from sklearn.preprocessing import OrdinalEncoder

encoder = OrdinalEncoder()
X[['education_encoded']] = encoder.fit_transform(X[['education']])

‚ö†Ô∏è Don't use this on nominal (unordered) data ‚Äî the model may assume mathematical relationship that doesn‚Äôt exist.

üî∑ 2. Label Encoding

  • Converts categories into integer values
  • Used for target/label column in classification (not features)
from sklearn.preprocessing import LabelEncoder

le = LabelEncoder()
y = le.fit_transform(y)

Example:

['dog', 'cat', 'dog', 'fish'] ‚Üí [1, 0, 1, 2]

‚ö†Ô∏è For input features, LabelEncoding is dangerous ‚Äî use OneHotEncoding instead for unordered variables.

‚úÖ Day 27: One Hot Encoding (OHE)

üî≥ What it does:

Creates binary columns for each category.

Example:

Color ‚Üí ['Red', 'Blue', 'Green']
OHE ‚Üí [1,0,0], [0,1,0], [0,0,1]

from sklearn.preprocessing import OneHotEncoder
ohe = OneHotEncoder(sparse=False, drop='first')
X_encoded = ohe.fit_transform(X[['color']])

üß† Dummy Variable Trap:

  • Happens when OHE creates multicollinearity
  • Fix: use drop='first' to remove 1 column

‚úÖ When to use OHE vs Ordinal:

Data Type Encoding
Ordered (e.g. Low < Medium < High) Ordinal
Unordered (e.g. City, Gender) One Hot Encoding

‚úÖ Day 28: ColumnTransformer in Sklearn

üéØ What is it?

Efficient way to apply different preprocessing steps to different columns.

üîß Example Use Case:

from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import StandardScaler, OneHotEncoder

ct = ColumnTransformer(
    transformers=[
        ('num', StandardScaler(), ['age', 'salary']),
        ('cat', OneHotEncoder(drop='first'), ['gender', 'city'])
    ],remainder='passthrough'
)

X_processed = ct.fit_transform(X)
  • 'num' applies StandardScaler to numerical features
  • 'cat' applies OneHotEncoder to categorical features

üîó Why it‚Äôs useful:

  • Keeps preprocessing organized
  • Works smoothly inside pipelines
  • Handles mixed-type datasets easily

‚úÖ Bonus: Common Pitfalls to Avoid

Mistake Fix
Using LabelEncoder on features Use OHE or Ordinal depending on context
Not scaling after encoding Apply StandardScaler on numeric columns only
Manual encoding on large data Use ColumnTransformer to automate and chain steps

‚úÖ Day 29: Machine Learning Pipelines A‚ÄìZ

üéØ Why Pipelines?

  • Chains multiple preprocessing and modeling steps into one object
  • Makes code clean, reusable, and deployable

üîß Example:

from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler

    ('scaler', StandardScaler()),
    ('model', LinearRegression())
])

üîó Real Use with ColumnTransformer:

from sklearn.compose import ColumnTransformer
from sklearn.pipeline import Pipeline
from sklearn.impute import SimpleImputer
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.linear_model import LogisticRegression

numeric_pipeline = Pipeline([
    ('imputer', SimpleImputer(strategy='mean')),
    ('scaler', StandardScaler())
])

categorical_pipeline = Pipeline([
    ('imputer', SimpleImputer(strategy='most_frequent')),
    ('ohe', OneHotEncoder(drop='first'))
])

preprocessor = ColumnTransformer([
    ('num', numeric_pipeline, ['age', 'salary']),
    ('cat', categorical_pipeline, ['gender', 'city'])
])

pipeline = Pipeline([
    ('preprocess', preprocessor),
    ('model', LogisticRegression())
])

üìÇ Handling Missing Values (Numerical & Categorical)

‚úÖ Day 36: Handling Missing Data (Numerical) ‚Äì SimpleImputer

üîç Problem:

  • Missing numerical values (NaN) can break models.

üõ†Ô∏è Strategy:

Use SimpleImputer to fill missing values using:

  • mean
  • median
  • most_frequent
  • constant

üîß Example:

from sklearn.impute import SimpleImputer

imputer = SimpleImputer(strategy='mean')  # or 'median'
X['age'] = imputer.fit_transform(X[['age']])

üîé When to Use:

Strategy When to Use
mean Symmetric, no outliers
median Skewed data or outliers
constant Business logic (e.g., fill salary = 0)

‚ö†Ô∏è Never impute using the mean on categorical variables!

‚úÖ Day 37: Handling Missing Data (Categorical)

üß∞ Strategy 1: Fill with Most Frequent Value

imputer = SimpleImputer(strategy='most_frequent')
X['city'] = imputer.fit_transform(X[['city']])

üß∞ Strategy 2: Fill with a New Category (like "Missing")

X['city'] = X['city'].fillna('Missing')
  • Useful when missingness carries meaning (e.g., no education ‚Üí maybe unemployed)

‚úÖ Bonus: Add a Missing Indicator

X['city_missing_flag'] = X['city'].isnull().astype(int)

Helps models learn if "missingness" itself is predictive.

üìå 2. Random Sample Imputation

import random

def random_sample_impute(df, variable):
    df[variable + "_random"] = df[variable]
    random_sample = df[variable].dropna().sample(df[variable].isnull().sum(), random_state=0)
    random_sample.index = df[df[variable].isnull()].index
    df.loc[df[variable].isnull(), variable + "_random"] = random_sample

üöÄ Keeps data distribution unchanged (unlike mean imputation)

‚úÖ Day 41: What are Outliers?

üìå Definition:

Outliers are data points that deviate significantly from other observations.

Type Examples
Global Income = ‚Çπ1 Cr in a dataset where average = ‚Çπ30k
Contextual Height = 5‚Äô2 is normal for females, short for males
Collective Multiple incorrect values due to sensor failure

üîç Why handle them?

  • They distort the mean, variance, correlation
  • Can negatively impact many ML models (especially linear models)

‚úÖ Techniques to detect outliers:

  1. Boxplot
  2. Z-score method
  3. IQR method
  4. Domain knowledge

‚úÖ Day 42: Outlier Detection ‚Äì Z-Score Method

‚öôÔ∏è Z-Score:

Formula:
                 \( Z = \frac{(x - \mu)}{\sigma} \)

  • A Z-score > 3 or < -3 is typically an outlier

üîß Code:

from scipy import stats
import numpy as np

z = np.abs(stats.zscore(df['salary']))
df_clean = df[z < 3]

‚ö†Ô∏è When to use:

  • Data should be normally distributed
  • Not robust to extreme outliers

‚úÖ Day 43: Outlier Detection ‚Äì IQR Method

‚öôÔ∏è IQR = Interquartile Range

Formula:
                 \( IQR = Q3 - Q1 \)

Outlier Range:
                 \( \text{Lower} = Q1 - 1.5 \times IQR \)
                 \( \text{Upper} = Q3 + 1.5 \times IQR \)

üîß Code:

Q1 = df['salary'].quantile(0.25)
Q3 = df['salary'].quantile(0.75)
IQR = Q3 - Q1

lower = Q1 - 1.5 * IQR
upper = Q3 + 1.5 * IQR

df_clean = df[(df['salary'] >= lower) & (df['salary'] <= upper)]

üß† IQR is better than Z-score for skewed data and more robust.

‚úÖ Day 44: Percentile Method + Winsorization

üìå Percentile Method:

Drop rows outside 1st and 99th percentile.

lower = df['salary'].quantile(0.01)
upper = df['salary'].quantile(0.99)
df = df[(df['salary'] > lower) & (df['salary'] < upper)]

üõ†Ô∏è Winsorization:

Cap values instead of removing.

import pandas as pd
from feature_engine.outliers import Winsorizer

winsor = Winsorizer(capping_method='iqr', tail='both', fold=1.5, variables=['salary'])
df = winsor.fit_transform(df)
  • Keeps row count intact
  • Safer for small datasets

üìâ Dimensionality Reduction + Regression Kickoff

‚úÖ Day 46: Curse of Dimensionality

üéØ What is it?

As the number of features (dimensions) increases:

  • Data becomes sparse
  • Distance-based algorithms fail
  • Overfitting increases

üìä Example:

In 2D: distance is meaningful. In 100D: all distances become almost equal, losing meaning.

üí• Effects:

Problem Caused by
Overfitting Too many irrelevant features
Computational cost More features = more complexity
Poor generalization Models memorize noise

‚úÖ Solution:

Use Dimensionality Reduction like PCA, feature selection, etc.

üí° Always remove low-variance or irrelevant features before modeling.

‚úÖ Day 47: PCA ‚Äì Principal Component Analysis (Part 1: Geometric Intuition)

üéØ What is PCA?

A linear technique to reduce feature space by projecting data to new axes (principal components) that:

  • Capture maximum variance
  • Are orthogonal (uncorrelated)

üìà Idea:

  • Rotate coordinate axes
  • Drop least-informative dimensions (PC3, PC4, etc.)
  • Keep only top-K components

üìå Geometrically:

PCA finds a new "viewpoint" where:

  • 1st component = direction of highest variance
  • 2nd component = next most variance, perpendicular to 1st

‚úÖ Day 48: PCA ‚Äì Mathematical Steps (Part 2)

üìê PCA Step-by-Step:

Standardize Data:
                 \( z = \frac{x - \mu}{\sigma} \)

Compute Covariance Matrix:
                 \( \text{Cov}(X) = \frac{1}{n-1} (X^T \cdot X) \)

  1. Find Eigenvalues & Eigenvectors

  2. Sort Eigenvectors by Eigenvalue (variance explained)

  3. Project Data onto top K eigenvectors

Project Data onto New Feature Space:
                 \( Z = X \cdot W_k \)

üìà Explained Variance Ratio (EVR):

Tells how much variance each component explains

from sklearn.decomposition import PCA

pca = PCA()
pca.fit(X_scaled)
print(pca.explained_variance_ratio_)

‚úÖ Day 49: PCA ‚Äì Code Implementation (Part 3)

üîç Choose n_components using:

  • Scree plot (elbow method)
  • Cumulative EVR ‚â• 95%

‚úÖ Day 51: Simple Linear Regression ‚Äì Mathematical Formulation

üéØ Goal:

Predict target y from input feature x using:

Predicted Value (Linear Regression):
                 \( \hat{y} = mx + c \)

‚úÖ Key Concepts:

üìê 1. Loss Function:

  • Mean Squared Error (MSE)

Mean Squared Error (MSE):
                 \( MSE = \frac{1}{n} \sum (y_i - \hat{y}_i)^2 \)

üìê 2. Optimization Objective:

Minimize MSE by finding best m and c

üìê 3. Gradient Descent (introduction):

Used to find optimal m, c when dataset is large or no closed-form solution is desired.

‚úÖ Day 52: Regression Metrics (MSE, MAE, RMSE, R¬≤ Score)

üìä 1. Mean Absolute Error (MAE)

Mean Absolute Error (MAE):
                 \( MAE = \frac{1}{n} \sum |y_i - \hat{y}_i| \)

  • Easy to interpret
  • Robust to outliers

üìä 2. Mean Squared Error (MSE)

Mean Squared Error (MSE):
                 \( MSE = \frac{1}{n} \sum (y_i - \hat{y}_i)^2 \)

  • Penalizes large errors

üìä 3. Root Mean Squared Error (RMSE)

Root Mean Squared Error (RMSE):
                 \( RMSE = \sqrt{MSE} \)

  • Same unit as target

üìä 4. R¬≤ Score (Coefficient of Determination)

R-Squared (Coefficient of Determination):
                 \( R^2 = 1 - \frac{SS_{\text{res}}}{SS_{\text{tot}}} \)

  • 0 ‚â§ R¬≤ ‚â§ 1
  • R¬≤ = % of variance explained by the model

üîß Sklearn Example:

from sklearn.metrics import mean_squared_error, mean_absolute_error, r2_score

mse = mean_squared_error(y_test, y_pred)
rmse = np.sqrt(mse)
mae = mean_absolute_error(y_test, y_pred)
r2 = r2_score(y_test, y_pred)

‚úÖ Day 53: Multiple Linear Regression ‚Äì Intuition + Code

üéØ Formula:

Multiple Linear Regression Equation:
                 \( \hat{y} = b_0 + b_1x_1 + b_2x_2 + \cdots + b_nx_n \)

  • Each feature has its own coefficient
  • Line becomes a hyperplane in higher dimensions

üß† Real-World:

Used for predicting price, sales, etc. using multiple variables like area, location, age, etc.

üîß Code:

‚úÖ Day 54: Multiple Linear Regression ‚Äì Mathematical Formulation

üìê Matrix Form:

Linear Model (Matrix Form):
                 \( Y = XW + b \)

  • X: Feature matrix (n √ó m)
  • W: Weight vector (m √ó 1)
  • b: Bias (scalar or vector)

üéØ Closed-form solution (Normal Equation):

Normal Equation (Closed-form Solution for Linear Regression):
                 \( W = (X^TX)^{-1}X^Ty \)

  • Can be used when data is small
  • Fails if $X^TX$ is non-invertible
